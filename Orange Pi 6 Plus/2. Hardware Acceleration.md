# Hardware Acceleration

Configuring hardware acceleration on Orange Pi 6 Plus is tricky because mainstream support is still evolving. It is best to start from a customized OS image, which can be installed following a [dedicated guide](./1.%20Multiboot%20Setup.md).

## Ubuntu 24.04 Desktop Orange Pi 6 Plus Edition

Once you have a working system, run the following command to check if hardware acceleration enabled:

```sh
ls /dev/dri
```

Reference output:

```sh
by-path  card0  card1  card2  card3  card4  renderD128  renderD129  renderD130  renderD131  renderD132
```

This output means that Linux DRM subsystem detected several graphics-related devices.

## Gentoo Sky1 Linux edition

To enable hardware acceleration in gentoo, navigate to your home directory:

```sh
cd
```

Clone this project:

::: tabs

=== ssh

```sh
git clone git@github.com:zeionara/orange-pi-6-plus-gpu.git
```

=== https

```sh
git clone https://github.com/zeionara/orange-pi-6-plus-gpu.git
```

:::

Install pre-built kernel with `GPU` driver:

```sh
cd orange-pi-6-plus
./setup.sh
```

Add `menuentry` in `/boot/grub/grub.cfg` which looks like this (replace `a277cbdd-5225-44c9-92b1-d16956339a4f` in highlighted lines with `uuid` of your partition):

```sh{6,8}
menuentry "Sky1 Linux, with Linux 6.19.0-rc8-sky1-rc.r6 (GPU+NPU)" --class debian --class gnu-linux --class gnu --class os {
        load_video
        insmod gzio
        insmod part_gpt
        insmod ext2
        search --no-floppy --fs-uuid --set=root a277cbdd-5225-44c9-92b1-d16956339a4f
        echo "Loading Sky1 Linux 6.19.0-rc8-sky1-rc.r6 ..."
        linux /boot/vmlinuz-6.19.0-rc8-sky1-rc.r6 root=UUID=a277cbdd-5225-44c9-92b1-d16956339a4f ro loglevel=7 console=tty0 console=ttyAMA2,115200 efi=noruntime acpi=off clk_ignore_unused linlon_dp.enable_fb=1 linlon_dp.enable_render=0 fbcon=map:01111111 rootwait
        echo "Loading initial ramdisk ..."
        initrd /boot/initrd.img-6.19.0-rc8-sky1-rc.r6
        echo "Loading device tree ..."
        devicetree /boot/sky1-orangepi-6-plus.dtb
}
```

Reboot with this kernel, then configure `mesa` use flags:

```sh
echo 'media-libs/mesa vulkan' | sudo tee -a /etc/portage/package.use/mesa
echo '*/* VIDEO_CARDS: -* panthor panfrost' | sudo tee /etc/portage/package.use/00video-cards
```

Then install `mesa` and some utils:

```sh
sudo emerge --ask --verbose --update --deep --newuse media-libs/mesa media-libs/vulkan-tools x11-libs/libdrm
```

Reboot, make sure that hardware acceleration works now. Run command:

```sh
vulkaninfo | grep deviceName
```

Should output:


## Compiling custom linux kernel

::: warning Work in progress

This chapter is incomplete, hopefully I will return to developing it later

Go to the home folder:

```sh
cd /home/$USER
```

Clone these repos there:

```sh
git clone https://github.com/Sky1-Linux/linux-sky1.git
git clone https://github.com/zeionara/sky1-linux-build.git
```

Prepare kernel sources and resolve any conflicts if they appear manually by editing `.c` files and removing `.rej` files:

```sh
cd sky1-linux-build
./scripts/prepare-source.sh 6.18.12
```

Then generate kernel config, set at least `General setup > Local version` to avoid kernel version collisions:

```sh
cd build/linux-6.18.12
make menuconfig
```

As an alternative to `menuconfig`, call:

```sh
make oldconfig
```

This will ask you to fill new options missing values in current `.config` file. Build kernel:

```sh
make -j$(nproc)
```

When build is completed, install kernel modules:

```sh
sudo make modules_install
```

And install the kernel:

```sh
sudo make install
```

If `dracut` is configured, it will generate boot files automatically, and `grub` config will be updated. Then reboot to check that updated kernel works as expected.
