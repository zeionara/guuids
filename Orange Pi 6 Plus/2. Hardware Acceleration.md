# Hardware Acceleration

Configuring hardware acceleration on Orange Pi 6 Plus is tricky because mainstream support is still evolving. It is best to start from a customized OS image, which can be installed following a [dedicated guide](./1.%20Multiboot%20Setup.md).

## Verify hardware acceleration

Once you have a working system, run the following command to check if hardware acceleration enabled:

```sh
ls /dev/dri
```

Reference output:

```sh
by-path  card0  card1  card2  card3  card4  renderD128  renderD129  renderD130  renderD131  renderD132
```

This output means that Linux DRM subsystem detected several graphics-related devices.

## Collect the required software to support hardware acceleration

Create directory in which you will collect the required drivers:

```sh
mkdir /home/$USER/gpu-drivers
cd /home/$USER/gpu-drivers
```

## Configuring Gentoo

Enable mesa USE flags:

```sh
echo 'media-libs/mesa vulkan' | sudo tee -a /etc/portage/package.use/mesa
echo '*/* VIDEO_CARDS: -* zink' | sudo tee -a /etc/portage/package.use/00video-cards
```

Emerge the required packages:

```sh
sudo emerge --ask --verbose --update --deep --newuse media-libs/mesa media-libs/vulkan-tools x11-libs/libdrm
```

## Compiling tailor-made linux kernel

Go to the home folder:

```sh
cd /home/$USER
```

Clone these repos there:

```sh
git clone https://github.com/Sky1-Linux/linux-sky1.git
git clone https://github.com/zeionara/sky1-linux-build.git
```

Prepare kernel sources and resolve any conflicts if they appear manually by editing `.c` files and removing `.rej` files:

```sh
cd sky1-linux-build
./scripts/prepare-source.sh 6.18.12
```

Then generate kernel config, set at least `General setup > Local version` to avoid kernel version collisions:

```sh
cd build/linux-6.18.12
make menuconfig
```

As an alternative to `menuconfig`, call:

```sh
make oldconfig
```

This will ask you to fill new options missing values in current `.config` file. Build kernel:

```sh
make -j$(nproc)
```

When build is completed, install kernel modules:

```sh
sudo make modules_install
```

And install the kernel:

```sh
sudo make install
```

If `dracut` is configured, it will generate boot files automatically, and `grub` config will be updated. Then reboot to check that updated kernel works as expected.
