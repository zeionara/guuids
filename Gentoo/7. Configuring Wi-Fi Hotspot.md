# Configuring Wi-Fi Hotspot

::: info Resources

The latest version of this guide and all mentioned configuration files are available in [codeberg repo][vpnap].

:::

This instruction guides you on using a set of open-source network tools for running a `wifi` hotspot with proxying requests through `vpn` on top of your `Gentoo` system. This guide supposes that you've cloned the repo to the `/opt` folder like this (replace `zeio` with your username):

```sh
git clone https://codeberg.org/zeionara/vpnap.git /tmp/vpnap && sudo mv /tmp/vpnap /opt/vpnap && sudo chown -R zeio:zeio /opt/vpnap
```

The commands below are tested on `Orange Pi 6 Plus` with `6.18.12-gentoo-dist linux kernel`.

## Install dependencies

Before setting up the required services, configure and install some dependencies:

1. Enable `script` use flag for `dnsmasq` package:

```sh
echo 'net-dns/dnsmasq script' | sudo tee /etc/portage/package.use/dnsmasq
```

2. Install the following dependencies:

```sh
sudo emerge --ask net-wireless/hostapd net-dns/dnsmasq net-firewall/iptables
```

## Configure iptables

To forward traffic from one interface to another you have to configure `ip tables`, which is crucial for enabling `wifi` access point.


Normally we are interested in three interfaces here:  
- Physical interface on which you are running `wifi` hotspot - let's call it `lan_iface`;  
- Physical interface that is connected to the internet - let's call it `wan_iface`;
- Virtual interface that is used for `VPN` - let's call it `vpn_iface`. 

For example, in my setup, `wifi` hotspot is running on `wlu1`, internet connection is running on `wlan0`, and `vpn` is using `Meta` interface. In the following subsections you will have to replace `lan_iface`, `wan_iface` and `vpn_iface` with interface names applicable to your system.

### Create masquerade rule

To enable ip address replacement, configure `MASQUERADE` rule:

```sh
sudo iptables -t nat -A POSTROUTING -o wan_iface -j MASQUERADE
```

For instance, if you are connected to the internet via wireless connection on `wlan0`, this command takes the following form:

```sh
sudo iptables -t nat -A POSTROUTING -o wlan0 -j MASQUERADE
```

To make sure that the rule is enabled, list the `nat` table contents:

```sh
sudo iptables -t nat -L -v
```

You should see a line like this:

```sh
Chain POSTROUTING (policy ACCEPT 64600 packets, 16M bytes)
pkts bytes target     prot opt in     out     source               destination
54383 5986K MASQUERADE  all  --  any    wlan0   anywhere             anywhere
```

### Create rules for forwarding packets from lan to wan

Now you need to create a set of rules for forwarding packets from `lan` interface to `wan`, one rule for each output interface:

```sh
sudo iptables -A FORWARD -i lan_iface -o wan_iface -j ACCEPT
sudo iptables -A FORWARD -i lan_iface -o vpn_iface -j ACCEPT
```

To make sure that the rules are enabled, list the main table contents:

```sh
sudo iptables -L -v
```

You should see lines like these:

```sh
Chain FORWARD (policy ACCEPT 8352 packets, 1508K bytes)
pkts bytes target     prot opt in     out     source               destination
5091K 1110M ACCEPT     all  --  wlu1   Meta    anywhere             anywhere
30111   13M ACCEPT     all  --  wlu1   wlan0   anywhere             anywhere
```

### Create rules for forwarding packets from wan to lan

In addition, you need to create a set of rules for forwarding packets backwards - from `wan` interface to `lan` for established connections, one rule for each input interface:

```sh
sudo iptables -A FORWARD -i wan_iface -o lan_iface -m state --state RELATED,ESTABLISHED -j ACCEPT
sudo iptables -A FORWARD -i vpn_iface -o lan_iface -m state --state RELATED,ESTABLISHED -j ACCEPT
```

To make sure that the rules are enabled, list the main table contents:

```sh
sudo iptables -L -v
```

You should see lines like these:

```sh
Chain FORWARD (policy ACCEPT 8352 packets, 1508K bytes)
pkts bytes target     prot opt in     out     source               destination
13M   15G ACCEPT     all  --  Meta   wlu1    anywhere             anywhere             state RELATED,ESTABLISHED
20546   20M ACCEPT     all  --  wlan0  wlu1    anywhere             anywhere             state RELATED,ESTABLISHED
```

### Persist changes

To keep the changes after reboot, save `iptables` state:

```sh
sudo rc-service iptables save
```

And make sure `ipatables` service is added to the `default` runlevel:

```sh
sudo rc-update add iptables default
```

## Configure services

1. Enable `ip` forwarding for current session:

```sh
sudo sysctl -w net.ipv4.ip_forward=1
```

And for future sessions:

```sh
echo 'net.ipv4.ip_forward=1' | sudo tee /etc/sysctl.conf
```

2. Configure `ip address` range for `dnsmasq`. Since in this guide we are using `192.168.1.1/24` as `wifi` hotspot `ip address`, we have to keep the same subnetwork for connected devices:

```sh
sudo tee -a /etc/dnsmasq.conf > /dev/null << EOF
interface=wlu1
dhcp-range=192.168.1.10,192.168.1.100,12h
EOF
```

3. If you are using `realtek` wifi dongles, create `switch-realtek-wifi-dongle-mode` service:

```sh
sudo cp /opt/vpnap/switch-realtek-wifi-dongle-mode /etc/init.d/
```

4. Add the service to `boot` runlevel - it needs to run before `wpa_supplicant` if your are going to use the interface as an access point to the Internet:

```sh
sudo rc-update add switch-realtek-wifi-dongle-mode boot
```

5. Replace `wan_iface` with interface name which you are using for accessing the Internet and copy file with `wpa_supplicant` args to `/etc/conf.d`:

```sh
sudo cp /opt/vpnap/wpa_supplicant /etc/conf.d/
```

6. Replace `network` section in `wpa_supplicant.conf` with output from command `wpa_passphrase <ssid> <psk>` and copy `wpa_supplicant.conf` to `/etc/wpa_supplicant`:

```sh
sudo cp /opt/vpnap/wpa_supplicant.conf /etc/wpa_supplicant/
```

7. Add `wpa_supplicant` to `default` runlevel:

```sh
sudo rc-update add wpa_supplicant default
```

8. Copy `hostapd` to `/etc/conf.d`:

```sh
sudo cp /opt/vpnap/hostapd /etc/conf.d/
```

9. Replace `lan_iface` with interface name which you are using for running `wifi` hotspot and copy `hostapd.conf` to `/etc/hostapd`:

```sh
sudo cp /opt/vpnap/hostapd.conf /etc/hostapd/
```

10. Replace `lan_iface` with interface name which you are using for running `wifi` hotspot, and create `wifi-ap` service:

```sh
sudo cp /opt/vpnap/wifi-ap /etc/init.d/
```

11. Add the service to `default` runlevel:

```sh
sudo rc-update add wifi-ap default
```

12. Create `vpn-ap` service:

```sh
sudo cp /opt/vpnap/vpn-ap /etc/init.d/
```

13. Create `vpn-ap` config file if needed:

```sh
sudo tee /etc/conf.d/vpn-ap >/dev/null <<EOF
vpn_ap_args="-d /home/zeio/.config/mihomo"
EOF
```

If you are using `mihomo`, the config folder `/home/zeio/.config/mihomo` should contain at least `config.yaml`, `GeoIP.dat`, and `GeoSite.dat` (the last two might be optional depending on your configuration).

14. Add `vpn-ap` to `default` runlevel:

```sh
sudo rc-update add vpn-ap default
```

## Troubleshoot

To manually start `wifi` hotspot try running the following command:

```sh
sudo rc-service net.wlu1 restart; sleep 1; sudo rc-service dnsmasq restart; sleep 1; sudo rc-service hostapd restart; sleep 10; sudo ip addr add 192.168.1.1/24 dev wlu1
```
