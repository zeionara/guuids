# Running AI workloads

If you've installed `Gentoo` on `ARM` SoC, you can use this image to run `LLM`s (tested on `Orange Pi 6 Plus`):

```sh
docker pull public.ecr.aws/q9t5s3a7/vllm-arm64-cpu-release-repo:v0.16.0
```

Here is an example of `docker-compose.yml`:

```yaml
services:
  vllm:
    image: public.ecr.aws/q9t5s3a7/vllm-arm64-cpu-release-repo:v0.16.0
    privileged: true
    shm_size: 2gb
    container_name: vllm
    ports:
      - 8000:80
    volumes:
      - ${DOCKER_VOLUME_DIRECTORY:-.}/volumes/vllm/.cache:/root/.cache
    restart: unless-stopped
    entrypoint: |
      vllm serve Qwen/Qwen3-0.6B
        --reasoning-parser deepseek_r1
        --max-model-len 512
        --max-num-seqs 1
        --max-num-batched-tokens 256
        --enable-chunked-prefill
        --port 80
```

An example of script to call the inference:

```sh
#!/bin/bash

port=${2:-8000}
host=${3:-localhost}

if test -z "$1"; then
  echo Question is required
    exit 0
    fi

    curl http://$host:$port/v1/chat/completions \
      -H 'Content-Type: application/json' \
        -d "{\"model\": \"Qwen/Qwen3-0.6B\", \"messages\": [{\"role\": \"user\", \"content\": \"$1\"}], \"stream\": false}" -s | python -m json.tool --no-ensure-ascii
```

The script is called like this:

```sh
./ask.sh 'Who are you?'
```
